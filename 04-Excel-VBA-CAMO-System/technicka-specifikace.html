1 Architektura souboru

Systém je realizován jako jeden soubor ve formátu .xlsm (Excel Macro-Enabled Workbook), který obsahuje 7 datových listů a sadu VBA modulů.

Schéma propojení listů

Aircraft Master
→
Task Cards
→
Due List
Logbook
→
Aircraft Master
→
Component Control
AD/SB Tracking
→
Reports

Klíčové principy

  • Jeden soubor — vše v jednom .xlsm, žádné externí závislosti
  • Named Ranges — tabulky pojmenované pro snadné odkazování z VBA
  • ListObject (Table) — každý datový rozsah formátován jako Excel tabulka
  • Oddělení dat a logiky — výpočty ve VBA, data v listech
  • Komentovaný kód — každá procedura s popisem účelu a parametrů

3 VBA moduly a UserForms

Struktura VBA projektu

' === Struktura VBA modulů === Modules/ modMain ' Hlavní inicializace, globální proměnné modCalculations ' Výpočty remaining FH/FC/Days, Due List refresh modLogbook ' Import letových hodin, aktualizace Aircraft Master modReports ' Generování reportů (Fáze 2) modUtils ' Pomocné funkce (formátování, validace) UserForms/ frmFlightEntry ' Formulář pro zápis letu (datum, reg., FH, FC) frmTaskEntry ' Formulář pro přidání/editaci Task Card frmAircraftEntry ' Formulář pro správu letadel frmReportGen ' Volba a generování reportů (Fáze 2) Class Modules/ clsAircraft ' Třída reprezentující letadlo clsTaskCard ' Třída reprezentující údržbový úkon

Projde všechny Task Cards ' 2. Pro každý task zjistí aktuální FH/FC letadla ' 3. Vypočte remaining values ' 4. Aplikuje barevné kódování ' 5. Seřadí podle priority (červená → žlutá → zelená) End Sub Public Function CalcRemainingFH(taskRow As Long) As Double ' Vrátí zbývající FH do dalšího provedení úkonu ' = (Due FH) - (Current Total FH z Aircraft Master) End Function Public Function GetStatusColor(remaining As Variant, _ intervalType As String) As Long ' Vrátí barvu buňky dle prahových hodnot: ' FH: >100 = zelená, 50-100 = žlutá, <50 = červená ' Calendar: >90 dní = zelená, 30-90 = žlutá, <30 = červená End Function

modLogbook — Zápis letových hodin

Public Sub RecordFlight(flightDate As Date, _ regMark As String, _ flightHours As Double, _ flightCycles As Long, _ Optional note As String = "") ' 1. Validuje vstup (datum, reg. značka existuje, FH > 0) ' 2. Zapíše řádek do Logbook listu ' 3. Aktualizuje Total FH/FC v Aircraft Master ' 4. Volá RefreshDueList() pro přepočet End Sub

UserForm: frmFlightEntry

Uživatelsky přívětivý formulář pro zápis letu s těmito prvky:

  • ComboBox — výběr letadla z Aircraft Master (reg. značky)
  • DatePicker — výběr data letu
  • TextBox FH — nalétané hodiny (validace: číslo > 0)
  • TextBox FC — počet cyklů (validace: celé číslo > 0)
  • TextBox Poznámka — volitelné pole
  • Tlačítko Uložit — volá RecordFlight
  • Tlačítko Storno — zavře formulář bez uložení

4 Výpočetní logika a barevné kódování

Pravidla přepočtu Due List

Due List se přepočítává automaticky po každém zápisu letu. Logika je následující:

Typ intervalu Výpočet "Remaining" Příklad
FH Due_FH - Current_Total_FH Due: 5000 FH, Current: 4920 FH → Remaining: 80 FH
FC Due_FC - Current_Total_FC Due: 3000 FC, Current: 2850 FC → Remaining: 150 FC
Calendar Due_Date - Today() Due: 15.6.2026, Dnes: 30.3.2026 → Remaining: 77 dní
Combined Minimum ze všech applicable FH rem: 80, Cal rem: 77 dní → Status dle nejmenšího

Prahové hodnoty pro barevné kódování

Barva FH Remaining FC Remaining Dny Remaining Status text
Zelená > 100 > 100 > 90 OK
Žlutá 50 – 100 50 – 100 30 – 90 Due Soon
Červená < 50 < 50 < 30 Overdue / Critical

Poznámka: Konkrétní prahové hodnoty budou konfigurovatelné v budoucích verzích. Výchozí hodnoty vychází ze standardní praxe CAMO organizací.

5 Datový tok mezi listy

Sekvence po zápisu letu

1. UserForm
frmFlightEntry
→
2. Logbook
nový řádek
→
3. Aircraft Master
update FH/FC
→
4. Due List
RefreshDueList()

Detailní popis toku

  1. Uživatel otevře frmFlightEntry — vybere letadlo, zadá datum, FH a FC
  2. Validace vstupu — VBA ověří, že reg. značka existuje, FH > 0, datum není v budoucnosti
  3. Zápis do Logbook — nový řádek s detaily letu
  4. Aktualizace Aircraft Master — přičtení FH a FC k Total hodnotám, aktualizace data
  5. Přepočet Due List — automaticky se přepočtou všechny remaining hodnoty
  6. Vizuální aktualizace — barevné kódování se aktualizuje, řazení dle priority

6 Formát výstupu

Dodaný soubor bude mít následující vlastnosti:

  • Formát: .xlsm (Excel Macro-Enabled Workbook)
  • Kompatibilita: Microsoft Excel 2016 a novější (Windows i macOS)
  • VBA kód: plně komentovaný v angličtině
  • Ochrana: listy nebudou zamčeny (klient může upravovat strukturu)
  • Vzorová data: soubor bude obsahovat 3 testovací letadla pro demonstraci funkcionality
  • Dokumentace: list "README" s krátkým návodem k použití

7 Co NENÍ součástí dodávky

Výslovně vyloučeno

  • Airworthiness výpočetní logika — konkrétní intervaly údržby, pravidla pro AD compliance a specifické výpočty dodá klient. Zhotovitel implementuje pouze mechanismus přepočtu.
  • Propojení s externími systémy — žádné API integrace, webové služby ani propojení s EASA/CAA databázemi
  • Webová verze — systém je výhradně Excel-based
  • Víceuživatelský přístup — soubor je určen pro jednoho uživatele (nebo sdílení přes SharePoint/OneDrive bez souběžného přístupu)
  • Automatické aktualizace dat — veškeré vstupy zadává uživatel ručně nebo přes UserForms
  • Odpovědnost za rozhodnutí — zhotovitel neodpovídá za rozhodnutí učiněná na základě výstupů nástroje

Zahrnuto v dodávce

  • Funkční .xlsm soubor se všemi moduly dle fáze
  • Komentovaný VBA kód
  • Vzorová data pro testování
  • Stručný návod k použití (list README v souboru)
  • 14denní podpora po předání (opravy chyb)

8 Rozšiřitelnost

Systém je navržen s ohledem na budoucí rozšíření:

  • Modulární VBA — každý modul je nezávislý, lze přidávat nové bez zásahu do stávajících
  • Named Ranges — snadné přidání nových sloupců bez rozbití referencí
  • Konfigurovatelné prahy — barevné kódování lze upravit změnou konstant v modCalculations
  • Import/Export — struktura dat umožňuje budoucí CSV import/export
  • Přechod na databázi — tabulková struktura je kompatibilní s migrací do SQL databáze

Možná budoucí rozšíření (mimo aktuální scope)

  • Power Query import z externích zdrojů
  • Automatické e-mailové notifikace (Outlook VBA)
  • Dashboard s grafy trendů náletů
  • Multi-file architektura pro větší flotily
  • Migrace na webovou aplikaci (Django + PostgreSQL)
